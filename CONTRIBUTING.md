# 贡献指南

感谢您有兴趣为 LLM QLoRA Fine-tuning Framework 做出贡献！本文档旨在为您提供清晰的指引，帮助您轻松地参与项目贡献。

## 行为准则

请遵守我们的行为准则，共同创建一个友好的社区环境。

## 开发环境设置

1. Fork 仓库
2. 克隆您的 forked 仓库：
   ```bash
   git clone https://github.com/<your-username>/llm-qlora-finetune-framework.git
   cd llm-qlora-finetune-framework
   ```
3. 安装依赖：
   ```bash
   pip install -r requirements.txt
   ```
4. 创建新分支：
   ```bash
   git checkout -b feature/your-feature-name
   ```

## 贡献类型

您可以为项目做出以下类型的贡献：

- 新功能开发
- Bug 修复
- 文档改进
- 测试用例编写
- 性能优化
- 支持新模型架构

## 提交 PR 的步骤

1. 确保您的代码符合项目编码风格
2. 测试您的更改
3. 更新相关文档
4. 提交 PR 并描述更改内容

## 代码规范

- 遵循 PEP 8 Python 编码规范
- 为新增功能添加适当的注释
- 保持代码简洁易读

## 提交信息规范

使用约定式提交格式：

```
feat: 新增功能
fix: 修复 bug
docs: 文档更新
style: 代码格式调整
refactor: 重构代码
test: 测试相关
chore: 其他不重要更新
```

## 问题报告

当您发现 bug 或需要新功能时，请提交 issue 并提供详细信息。

## 联系方式

如有疑问，请通过 GitHub issue 联系我们。
